# Plan: Implement Timeframe Toggle for Store Dashboard Stat Cards

## Goal
Enable individual timeframe toggling for bill and visit cards in StoreDashboardStatCardsBoard. Clicking a card cycles through: today → last 7 days → last 31 days → today.

## Current State Analysis
- StoreDashboardStatCardsBoard currently uses hardcoded values ("R123.47" for bill, "500k" for visits)
- Single global timeframe state affects all cards
- No backend APIs for fetching bill/visit data by timeframe
- Cards display static data without real data fetching

## Implementation Plan

### Phase 1: Backend APIs for Timeframe Data
**Files to modify:**
- `backend/controllers/StoreDashboardController.js` - Add new endpoints
- `backend/routes/storeDashboardRoutes.js` - Add new routes
- `backend/models/VisitLogModel.js` - Create model for visit tracking (if not exists)

**New endpoints needed:**
- `GET /api/dashboard/:storeId/bill?timeframe=today|week|month` - Get bill amount for timeframe
- `GET /api/dashboard/:storeId/visits?timeframe=today|week|month` - Get visit count for timeframe

**VisitLogModel structure:**
```javascript
{
  storeId: ObjectId (ref: Store),
  visitorId: String,
  ip: String,
  timestamp: Date,
  source: String
}
```

### Phase 2: Frontend Data Fetching
**Files to modify:**
- Create `frontend/src/features/dashboard/dashboardSlice.ts` - Redux slice for dashboard data
- Add async thunks for fetching bill and visit data by timeframe

**New Redux state structure:**
```typescript
interface DashboardState {
  billData: {
    today: number;
    week: number;
    month: number;
  };
  visitData: {
    today: number;
    week: number;
    month: number;
  };
  loading: boolean;
  error: string | null;
}
```

### Phase 3: Individual Card Timeframes
**Files to modify:**
- `frontend/src/components/store_dashboard/overview/StoreDashboardStatCardsBoard.tsx`

**Changes needed:**
- Remove global `timeframe` state
- Add individual timeframe state for bill and visits cards
- Update `cycleTimeframe` to work per card
- Pass different timeframe props to bill vs visits cards
- Fetch data based on individual card timeframes

**New card structure:**
```typescript
const cards = [
  {
    title: "Bill",
    timeframe: billTimeframe, // individual state
    onTimeframeChange: () => setBillTimeframe(cycleTimeframe),
    // ... other props
  },
  {
    title: "Total Visits", 
    timeframe: visitsTimeframe, // individual state
    onTimeframeChange: () => setVisitsTimeframe(cycleTimeframe),
    // ... other props
  }
];
```

### Phase 4: Data Integration
**Files to modify:**
- Update StoreDashboardStatCardsBoard to use real data from Redux
- Handle loading states and error handling
- Format currency and numbers properly

**Data flow:**
1. Component mounts → dispatch fetchDashboardData(storeId)
2. User clicks card → update local timeframe state → dispatch fetch specific metric
3. Redux updates → component re-renders with new data

### Phase 5: UI/UX Improvements
**Files to modify:**
- `frontend/src/components/store_dashboard/cards/StoreDashboardOverviewStatCard.tsx`

**Enhancements:**
- Better visual feedback when timeframe changes
- Loading indicators during data fetch
- Smooth transitions between values
- Clear timeframe labels ("Today", "7 Days", "30 Days")

## Technical Considerations

1. **Performance**: Cache timeframe data to avoid excessive API calls
2. **Error Handling**: Graceful fallbacks for failed API calls
3. **Data Freshness**: Consider auto-refresh intervals or manual refresh buttons
4. **Mobile UX**: Ensure swiper cards work well with individual timeframes
